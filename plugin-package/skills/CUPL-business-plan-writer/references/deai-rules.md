# Step 4.6: 中文去 AI 痕迹化规则

> 此文件包含 12 条中文专项去痕迹规则和质量检查表。在 Step 4.6 执行时，先调用 humanizer-zh 处理关键章节，然后无论 humanizer-zh 是否可用，都必须逐条执行以下规则。

---

## 12 条中文专项去痕迹规则

1. **破折号严格限制**：全文中连续两个破折号（——）不超过 5 处。这是 LLM 中文输出最明显的指纹——人类写中文很少频繁使用破折号。多余的破折号改为逗号、句号或分号。

2. **双引号限制**：除直接引用他人原话外，不使用双引号做强调。LLM 惯用的 "赋能"、"闭环"、"打法"、"私域" 等带引号的行业黑话，直接写成不加引号的自然表述。如需强调，用加粗或直接陈述。

3. **禁用 AI 标志词和句式**：检查并重写以下 LLM 高频模式——

   **过渡词和宣告句式：**
   - "值得注意的是" / "需要指出的是" / "应该看到" → 直接陈述，不预告
   - "综上所述" / "总而言之" / "总的来说" → 自然收尾，不宣告
   - "不可否认的是" / "毋庸置疑" / "毫无疑问" → 删除，直接说事实
   - "在这一背景下" / "从某种角度来说" / "从某种程度上" → 简化或删除

   **虚张声势的学术腔：**
   - "提供了有力支撑" → 用具体数据替代
   - "发挥着至关重要的作用" → 说明具体作用是什么
   - "具有重要的现实意义" → 说明对谁、在什么场景下有意义
   - "为进一步研究奠定了基础" → 如果是课程作业，这话轮不到你说

   **LLM 最爱的高频句式：**
   - "不仅……而且……" → 控制在全文 3 次以内
   - "从……到……再到……" → 全文最多 1 次
   - "一方面……另一方面……" → 全文最多 1 次
   - "首先……其次……最后……" → 全文最多 2 次（结构本身没问题，但 AI 用得太频繁了）
   - "可以说" / "应该说" / "必须承认" → 删除，这些是口语水词

4. **句式多样性**：连续 3 个句子不能以相同结构开头。检查 "通过……"、"随着……"、"在……下"、"根据……"、"由于……"、"为了……" 等介词开头，AI 最爱连续用。用短句打破节奏——偶尔用一个 5-8 字的干脆短句跟在长句后面，效果拔群。

5. **具体优于抽象**：用数字和事实替代模糊形容词。"市场很大" → "2025 年市场规模达 142.7 亿元"；"用户增长快" → "月活用户 3 个月内增长 230%"。如果一个陈述没有数字支撑，要么补数字，要么删掉。

6. **删除"水词"**：以下词除非有明确定义和必要，否则删除——
   "全方位"、"多层次"、"立体化"、"系统性"、"一站式"、"全链路"、"矩阵式"、"颗粒度"、"底层逻辑"、"顶层设计"、"差异化竞争"（没有差异化就没有竞争，这句话是废话）、"深耕"（99% 时候可以用"专注"替代）、"打造"（"建设"或"建立"更自然）、"赋能"（说清楚到底帮用户做了什么）

7. **专业术语自然化**：不要一上来就堆术语。先用人话解释概念，再在括号里给出术语。例如：先写 "平台从每笔交易中抽取一定比例的佣金"，再写 "(take rate)"。

8. **中文标点规范**：中文正文使用全角标点（，。！？）；引号使用中文引号（""）；数字与中文之间不加空格；英文单词前后加半角空格。

9. **段落结构多样化**：AI 写的段落几乎全是「总-分-总」结构——开头一句总结、中间展开、结尾再来一句总结。全文不能每段都是这个结构。部分段落可以用「问题-回答」结构（提一个问题，用整段回答），部分用「观点-反驳-修正」结构（先给常见看法、再说为什么不准、最后给出更准确的判断）。

10. **段落长度有起伏**：AI 倾向于每段 3-5 句话，像流水线出来的。人工检查时，确保有 1-2 句的短段落（比如单独一句强调一个关键数字），也有 8-10 句的长段落（需要展开复杂逻辑时）。一眼扫过去段落长短不一才对。

11. **语调分章节校准**：AI 的默认语调通篇一致，但不同章节应该有不同的语气——
    - 执行摘要：自信、简洁、有冲击力。每一句都在说服读者继续读。
    - 行业分析：冷静、客观、数据导向。像研究员在写报告，不像销售在推销。
    - 财务计划：保守、审慎、留有安全边际。不要用"乐观预计"这种词。
    - 风险分析：坦诚、不回避。最怕什么就写什么，然后写你怎么应对。

12. **朗读测试**：每改完一章，把它读出声来（或在脑子里默读）。如果你读到一个地方感觉"正常人不会这么说"——那就改。AI 写的文字有一个特征：看的时候觉得还行，读出来立刻觉得别扭。相信你的朗读直觉。

## 去痕迹质量检查

随机抽取 HTML 中 5 段正文（每段 200 字以上，覆盖不同章节），用以下标准逐段评分：

| 检查项 | 标准 | 不通过措施 |
|--------|------|-----------|
| 破折号 | 全文 ≤ 5 处 | 逐处改为逗号或句号 |
| 句式多样性 | 无连续 3 句相同开头 | 打破句式，插入短句 |
| 标志词 | 无连续 5 个以上所列词 | 逐词替换 |
| 段落结构 | 不是每段都「总-分-总」 | 改 2-3 段为其他结构 |
| 段落长度 | 有 <3 句的短段，也有 >6 句的长段 | 合并或拆分 |
| 朗读感 | 读出声来自然流畅，不机械 | 重写读起来别扭的句子 |
| 水词 | 无第 6 条所列词（除非必要） | 删除或替换 |

不通过则重新修订对应段落，直到全部通过。

## 向用户展示

将去痕迹后的关键段落（至少：执行摘要 + 2 段产品描述 + 1 段风险分析）展示给用户确认。询问："这些文字读起来自然吗？还有 AI 味吗？如果有，具体是哪句话让你觉得是 AI 写的？"
